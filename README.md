# TOPTW — Team Orienteering Problem with Time Windows

Курсова робота з дослідження операцій (НТУУ КПІ, ФІОТ, кафедра ІСТ).

**Задача:** складання портфеля з `k` туристичних маршрутів з часовими вікнами. Мета — максимізувати сумарну привабливість відвіданих об’єктів з урахуванням зменшення внеску при повторному включенні об’єкта в різні маршрути. Якщо об’єкт включено до `q` маршрутів, його внесок дорівнює `a + a*alpha + ... + a*alpha^(q-1)`.

## Алгоритми

- **Жадібний алгоритм** — послідовно формує маршрути, обираючи допустимий об’єкт з найбільшим відношенням маржинальної привабливості `a_i * alpha^q_i` до додаткових часових витрат.
- **Генетичний алгоритм** — steady-state ГА: кросовер переносить цілі маршрути від батьків, мутація виконує додавання, видалення або заміну об’єкта, після чого застосовується repair-процедура. Умова завершення — `k_no` ітерацій без покращення рекорду.

## Формат JSON-файлу задачі

```json
{
  "n": 5,
  "k": 2,
  "alpha": 0.7,
  "a": [8, 6, 7, 9, 5],
  "t": [30, 20, 25, 35, 15],
  "o": [0, 0, 60, 0, 120, 0],
  "c": [999, 480, 540, 420, 540, 660],
  "T": [120, 120],
  "s": [0, 0],
  "m": [3, 3],
  "d": [[0, 15, 20, 10, 25, 12], [15, 0, 10, 12, 15, 8]]
}
```

Матриця `d` у реальному файлі повинна мати розмір `(n+1) x (n+1)`. Індекс `0` відповідає офісу/депо, об’єкти мають індекси `1..n`.

## Меню програми

```
1. Внести дані задачі       — вручну / генерація / з JSON-файлу
2. Розв'язати задачу        — жадібний + генетичний алгоритми
3. Провести експерименти    — k_no / β / розмірність
4. Вивести дані задачі      — на екран / у файл
0. Завершити роботу
```

## Параметри генетичного алгоритму

| Параметр | Опис | Рекомендоване значення |
|----------|------|------------------------|
| `I` | Розмір популяції | 20 |
| `b` | Частка кращих особин Best для вибору батьків | 0.5 |
| `p_mut` | Імовірність мутації нащадка | 0.3 |
| `beta` | Частка найкращих кандидатів, з яких робиться випадковий вибір. Мале `beta` — більш жадібний вибір, велике `beta` — більша випадковість | 0.3 |
| `k_no` | Кількість ітерацій без покращення рекорду | 5*n або значення з експерименту |
| `A_max` | Максимальна кількість спроб генерації унікальної особини | 5 |
